## 4. Технические требования

### 4.1. Формат данных

Парсер должен создавать:

```csharp
// Узел с портами потока
NodeData {
    Id: "node_0",
    Type: NodeType.MathAdd,
    VariableName: "z",
    // Порты потока
    ExecIn: ...,
    ExecOut: ...,
    // Порты данных
    InputConnections: { "inputA": "node_x", "inputB": "node_y" },
    OutputConnections: { "output": "node_z" }
}
```

### 4.2. Пример парсинга цикла `for`

**Входной код:**
```csharp
for (int i = 0; i < 10; i++)
{
    sum += i;
}
```

**Ожидаемый результат:**
- Узел `ForNode` с портами:
  - `execIn` — вход потока
  - `init` — инициализация (`int i = 0`)
  - `condition` — условие (`i < 10`)
  - `increment` — инкремент (`i++`)
  - `body` — тело цикла (`sum += i`)
  - `execOut` — выход потока

### 4.3. Пример парсинга `Console.WriteLine`

**Входной код:**
```csharp
Console.WriteLine("Hello");
```

**Ожидаемый результат:**
- Узел `ConsoleWriteLineNode` с портами:
  - `execIn` — вход потока
  - `message` — сообщение для вывода
  - `execOut` — выход потока

## 5. Критерии готовности

| № | Критерий |
|---|----------|
| 1 | Ошибка сохранения (Vector2) исправлена |

| 3 | Циклы `for` и `while` парсятся в соответствующие ноды |
| 4 | `Console.WriteLine` парсится в отдельную ноду |
| 5 | Все узлы имеют порты потока выполнения (`execIn`, `execOut`) |
| 6 | Условные узлы имеют порты `true`/`false` |
| 7 | Циклы имеют порты `init`, `condition`, `increment`, `body` |
| 8 | Нет ошибок компиляции |
| 9 | Парсер проходит тесты Кирилла |
